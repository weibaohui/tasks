# 任务闭环追踪系统 - 需求与详细设计

## 一、需求概述

### 1.1 核心功能

建立一套**分层任务分发与闭环追踪**机制：
- 任务可** spawn 子任务**，形成树状结构
- 子任务执行中**定时上报进度**
- 父任务维护**Todo列表**，实时感知子任务完成状态
- 形成 **spawn → 执行 → 进度上报 → Todo更新** 的闭环

### 1.2 业务流程

```
1. 启动任务 (StartTask) ────────────────────────────── 同步等待 10秒
   │
   ├─ 90% 概率: 直接完成
   │     └─ Finish(taskID) → 父任务完成
   │
   └─ 10% 概率: 分发子任务
         │
         ├─ StartSubTask(taskID, goal, taskType) × N
         │    └─ → 立即返回 subTaskID（不等待子任务执行）
         │
         ├─ publishTodo(taskID, [...])  → 立即展示 Todo 列表
         │    └─ 前端立即看到所有子任务（状态: distributed）
         │
         ├─ [子任务-1] 后台并行执行中... ─────────────────┐
         ├─ [子任务-2] 后台并行执行中... ─────────────────┼─ WorkerPool
         └─ [子任务-3] 后台并行执行中... ─────────────────┘
               │
               ├─ Process(subTaskID, 10%)  // 每10秒上报进度
               ├─ Process(subTaskID, 20%)  // 前端实时看到进度更新
               ├─ ...
               └─ Finish(subTaskID)
                         │
                         └─ 父任务收到 Finish 事件
                              └─ 更新 Todo 中该 subTaskID → "completed"
```

### 3.3 TraceContext (追踪上下文 - 核心)

```go
// 追踪上下文：管理整个 Trace 的追踪和级联取消
type TraceContext struct {
    TraceID    string                    // 全局唯一 TraceID
    RootTaskID string                    // 根任务 ID

    // Span 管理
    mu         sync.Mutex
    spans      map[string]*Span        // spanID -> Span
    spanCounter int                     // 用于生成自增 span 序号

    // 任务树（用于级联取消）
    taskTree   map[string][]string      // taskID -> [subTaskIDs]
    rootCtx    context.Context           // 根 context
    cancelFunc context.CancelFunc       // 根取消函数
}

type Span struct {
    SpanID       string   `json:"span_id"`
    ParentSpanID  string   `json:"parent_span_id,omitempty"`  // nil 表示根 Span
    TaskID       string   `json:"task_id"`
    Status       string   `json:"status"`
}

func (tc *TraceContext) GenerateSpanID(parentSpanID string) string {
    tc.mu.Lock()
    defer tc.mu.Unlock()
    tc.spanCounter++
    if parentSpanID == "" {
        return fmt.Sprintf("span-%03d", tc.spanCounter)
    }
    return fmt.Sprintf("%s-%03d", parentSpanID, tc.spanCounter)
}

func (tc *TraceContext) RegisterTask(taskID, parentTaskID, spanID, parentSpanID string) {
    tc.mu.Lock()
    defer tc.mu.Unlock()

    tc.spans[spanID] = &Span{
        SpanID:      spanID,
        ParentSpanID: parentSpanID,
        TaskID:      taskID,
        Status:      "running",
    }

    if parentTaskID != "" {
        tc.taskTree[parentTaskID] = append(tc.taskTree[parentTaskID], taskID)
    }
}

// CancelAll 取消整个 Trace 下所有任务（级联取消）
func (tc *TraceContext) CancelAll() {
    tc.mu.Lock()
    defer tc.mu.Unlock()

    if tc.cancelFunc != nil {
        tc.cancelFunc()
    }

    // 递归取消所有子任务
    tc.cancelTaskTree(tc.RootTaskID)
}

func (tc *TraceContext) cancelTaskTree(taskID string) {
    if children, ok := tc.taskTree[taskID]; ok {
        for _, childID := range children {
            // 通知后端取消子任务
            backend.CancelTask(childID)
            tc.cancelTaskTree(childID)
        }
    }
}
```

#### StartSubTask - 创建子任务 (内部 API)
```go
// 在 TaskContext 内部调用
// 先创建子任务（不立即更新 Todo）
// 自动传播 TraceID 和父子 Span 关系
func StartSubTask(ctx *TaskContext, goal string, taskType TaskType) *SubTaskResult {
    // 生成子任务的 SpanID（继承父 SpanID）
    spanID := ctx.TraceContext.GenerateSpanID(ctx.SpanID)

    // 创建子任务，自动继承 TraceID，携带父 SpanID
    subTask := backend.CreateSubTask(&SubTaskConfig{
        Goal:         goal,
        TaskType:     taskType,
        TraceID:      ctx.TraceContext.TraceID,  // 继承 TraceID
        ParentTaskID: ctx.TaskID,
        ParentSpanID: ctx.SpanID,                 // 记录父 SpanID
        SpanID:      spanID,
    })

    // 注册到 TraceContext（用于级联取消）
    ctx.TraceContext.RegisterTask(subTask.ID, ctx.TaskID, spanID, ctx.SpanID)

    // 记录到 TodoList（但不发布）
    ctx.TodoList.AddItem(subTask.ID, goal, taskType, spanID, "distributed")

    return &SubTaskResult{
        SubTaskID:    subTask.ID,
        SpanID:       spanID,
        ParentSpanID: ctx.SpanID,
    }
}
```

```go
// 子任务完成时调用

// 后端收到后:
1. 更新子任务状态为 completed
2. 查找父任务
3. 更新父任务 TodoList 中对应项状态为 completed
4. 发布 todo_updated 事件
5. 如果父任务的所有子任务都完成，父任务自动完成
```

### 4.3 级联取消 (Cascade Cancellation)

```go
// 取消任务时，级联取消所有子任务
CancelTask(taskID string) {
    // 1. 从 TaskRegistry 获取任务的 TraceContext
    tc := taskRegistry.GetTraceContext(taskID)

    // 2. 调用 TraceContext.CancelAll()
    tc.CancelAll()  // 会取消整个 Trace 下的所有任务
}

// TraceContext.CancelAll() 实现:
func (tc *TraceContext) CancelAll() {
    // 1. 调用根 context 的 cancelFunc
    if tc.cancelFunc != nil {
        tc.cancelFunc()
    }

    // 2. 递归取消任务树中的所有子任务
    tc.cancelTaskTree(tc.RootTaskID)
}

func (tc *TraceContext) cancelTaskTree(taskID string) {
    // 通知后端取消任务
    backend.CancelTask(taskID)

    // 递归取消所有子任务
    if children, ok := tc.taskTree[taskID]; ok {
        for _, childID := range children {
            tc.cancelTaskTree(childID)
        }
    }
}
```

## 五、执行器设计

### 5.1 任务执行器 (TaskExecutor)

```go
type TaskExecutor struct {
    taskRepo  domain.TaskRepository
    eventBus *bus.EventBus
    registry *TaskRegistry  // 注册所有运行中的任务上下文
}

func (e *TaskExecutor) ExecuteTask(ctx context.Context, task *domain.Task) {
    // 1. 创建任务执行上下文
    taskCtx := &TaskContext{
        TaskID:   task.ID().String(),
        Goal:     task.Metadata["goal"],
        TaskType: task.Type(),
        TodoList: e.loadOrCreateTodoList(task),
    }

    // 2. 注册上下文
    e.registry.Register(taskCtx)

    // 3. 执行任务逻辑
    e.runTaskLogic(taskCtx)

    // 4. 注销上下文
    e.registry.Unregister(taskCtx.TaskID)
}

func (e *TaskExecutor) runTaskLogic(ctx *TaskContext) {
    // sleep 10秒
    time.Sleep(10 * time.Second)

    // 90% 直接完成
    if rand.Float32() < 0.9 {
        Finish(ctx.TaskID)
        return
    }

    // 10% 分发子任务 (可指定不同类型)
    subTasks := []struct {
        goal      string
        taskType  TaskType
    }{
        {"处理前50%数据", TaskTypeDataProcessing},
        {"处理后50%数据", TaskTypeFileOperation},
        {"验证处理结果", TaskTypeAPICall},
    }

    // 先依次创建所有子任务，获得 N 个 subTaskID
    var subTaskIDs []string
    for _, st := range subTasks {
        subID := StartSubTask(ctx, st.goal, st.taskType)
        subTaskIDs = append(subTaskIDs, subID)
    }

    // 再一次性发布完整的 Todo 列表
    PublishTodo(ctx.TaskID, ctx.TodoList)

    // 等待所有子任务完成 (通过事件驱动)
}
```

### 5.2 事件处理

```go
// 订阅任务完成事件
eventBus.Subscribe(func(event DomainEvent) {
    if e, ok := event.(*SubTaskCompletedEvent); ok {
        // 更新父任务的 Todo 列表
        parentTask := e.GetParentTask()
```

## 七、Todo 列表更新规则

### 7.1 触发更新的事件

| 事件 | 更新内容 |
|------|---------|
| `StartSubTask()` 调用 | 添加新 item，status=distributed |
| `Process(subID, x%)` | 更新对应 item 的 progress |
| `Finish(subID)` | 更新 status=completed, progress=100 |

### 7.2 自动完成判断

当父任务满足以下条件时，自动调用 `Finish(parentID)`：
- 所有子任务 status = "completed"
- 父任务本身状态 = running
