# 任务闭环追踪系统 - 需求与详细设计

## 一、需求概述

### 1.1 核心功能

建立一套**分层任务分发与闭环追踪**机制：
- 任务可** spawn 子任务**，形成树状结构
- 子任务执行中**定时上报进度**
- 父任务维护**Todo列表**，实时感知子任务完成状态
- 形成 **spawn → 执行 → 进度上报 → Todo更新** 的闭环

### 1.2 业务流程

```
1. 启动任务 (StartTask)
   │
   ├─ sleep 10秒
   │
   ├─ 90% 概率: 直接完成
   │     └─ Finish(taskID) → 父任务 Todo 标记为 "completed"
   │
   └─ 10% 概率: 分发子任务
         │
         ├─ StartSubTask(taskID, goal, taskType) × N → 依次返回 N 个 subTaskID
         │
         ├─ publishTodo(taskID, [
         │     { subTaskID-1, goal-1, taskType-1, status: "distributed" },
         │     { subTaskID-2, goal-2, taskType-2, status: "distributed" },
         │     { subTaskID-3, goal-3, taskType-3, status: "distributed" }
         │   ])  // 一次性发布所有子任务
         │
         ├─ [子任务] Process(subTaskID, 10%)  // 每10秒
         ├─ [子任务] Process(subTaskID, 20%)
         ├─ ...
         └─ [子任务] Finish(subTaskID)
                    │
                    └─ 父任务收到 Finish 事件 → 更新 Todo 中该 subTaskID 状态为 "completed"
```

## 三、数据结构

### 3.1 TodoItem (Todo 列表项)

```go
type TodoItem struct {
    SubTaskID   string      `json:"sub_task_id"`    // 子任务ID
    SubTaskType TaskType   `json:"sub_task_type"`   // 子任务类型: data_processing | file_operation | api_call | custom
    Goal        string     `json:"goal"`            // 子任务目标/描述
    Status      TodoStatus `json:"status"`           // distributed | completed | failed
    Progress    int        `json:"progress"`         // 0-100
    CreatedAt   int64      `json:"created_at"`
    CompletedAt *int64     `json:"completed_at,omitempty"`
}

type TodoStatus string
const (
    TodoStatusDistributed TodoStatus = "distributed"  // 已分发
    TodoStatusCompleted   TodoStatus = "completed"    // 已完成
    TodoStatusFailed      TodoStatus = "failed"      // 失败
)
```

### 3.2 TodoList (Todo 列表)

```go
type TodoList struct {
    TaskID     string      `json:"task_id"`      // 父任务ID
    Items      []TodoItem `json:"items"`         // 子任务列表
    CreatedAt  int64       `json:"created_at"`
    UpdatedAt  int64       `json:"updated_at"`
}
```

### 3.3 TaskContext (任务执行上下文)

```go
type TaskContext struct {
    TaskID     string
    Goal       string           // 任务目标
    TaskType   TaskType       // 任务类型: data_processing | file_operation | api_call | custom
    TodoList   *TodoList      // 子任务列表
    OnProgress func(progress int)  // 进度回调
    OnComplete func()            // 完成回调
    OnSubTask  func(goal string, taskType TaskType) string  // 创建子任务回调
    Cancel     context.CancelFunc
}
```

## 四、API 设计

### 4.1 核心 API

#### StartTask - 启动任务
```
POST /api/v1/tasks/{id}/start

Request Body:
{
  "goal": "处理 10000 条数据"  // 可选，不传则使用默认执行逻辑
}

Response:
{
  "message": "task started"
}
```

#### StartSubTask - 创建子任务 (内部 API)
```go
// 在 TaskContext 内部调用
func StartSubTask(ctx *TaskContext, goal string, taskType TaskType) string {
    // 调用后端创建子任务，返回子任务ID
    subTaskID := backend.CreateSubTask(ctx.TaskID, goal, taskType)

    // 更新父任务的 TodoList
    ctx.UpdateTodo(subTaskID, goal, taskType, "distributed")

    return subTaskID
}
```

#### PublishTodo - 发布 Todo 列表 (事件)
```go
// 发布 Todo 列表更新事件
PublishTodo(taskID string, todoList *TodoList)

// 事件结构:
{
  "type": "todo_published",
  "task_id": "parent-123",
  "todo_list": {
    "task_id": "parent-123",
    "items": [
      {
        "sub_task_id": "child-456",
        "sub_task_type": "data_processing",
        "goal": "处理前 5000 条",
        "status": "distributed",
        "progress": 0
      }
    ]
  },
  "timestamp": 1234567890
}
```

#### Process - 报告进度 (内部 API)
```go
// 子任务每10秒调用
Process(subTaskID string, progress int)

// 后端收到后:
1. 更新子任务进度
2. 查找父任务
3. 更新父任务的 TodoList 中对应项的 progress
4. 发布 todo_updated 事件
```

#### Finish - 完成任务 (内部 API)
```go
// 子任务完成时调用
Finish(subTaskID string)

// 后端收到后:
1. 更新子任务状态为 completed
2. 查找父任务
3. 更新父任务 TodoList 中对应项状态为 completed
4. 发布 todo_updated 事件
5. 如果父任务的所有子任务都完成，父任务自动完成
```

## 五、执行器设计

### 5.1 任务执行器 (TaskExecutor)

```go
type TaskExecutor struct {
    taskRepo  domain.TaskRepository
    eventBus *bus.EventBus
    registry *TaskRegistry  // 注册所有运行中的任务上下文
}

func (e *TaskExecutor) ExecuteTask(ctx context.Context, task *domain.Task) {
    // 1. 创建任务执行上下文
    taskCtx := &TaskContext{
        TaskID:   task.ID().String(),
        Goal:     task.Metadata["goal"],
        TaskType: task.Type(),
        TodoList: e.loadOrCreateTodoList(task),
    }

    // 2. 注册上下文
    e.registry.Register(taskCtx)

    // 3. 执行任务逻辑
    e.runTaskLogic(taskCtx)

    // 4. 注销上下文
    e.registry.Unregister(taskCtx.TaskID)
}

func (e *TaskExecutor) runTaskLogic(ctx *TaskContext) {
    // sleep 10秒
    time.Sleep(10 * time.Second)

    // 90% 直接完成
    if rand.Float32() < 0.9 {
        Finish(ctx.TaskID)
        return
    }

    // 10% 分发子任务 (可指定不同类型)
    subTasks := []struct {
        goal      string
        taskType  TaskType
    }{
        {"处理前50%数据", TaskTypeDataProcessing},
        {"处理后50%数据", TaskTypeFileOperation},
        {"验证处理结果", TaskTypeAPICall},
    }

    for _, st := range subTasks {
        subID := StartSubTask(ctx, st.goal, st.taskType)
        ctx.TodoList.AddItem(subID, st.goal, st.taskType, "distributed")
    }

    // 发布 Todo 列表
    PublishTodo(ctx.TaskID, ctx.TodoList)

    // 等待所有子任务完成 (通过事件驱动)
}
```

### 5.2 事件处理

```go
// 订阅任务完成事件
eventBus.Subscribe(func(event DomainEvent) {
    if e, ok := event.(*SubTaskCompletedEvent); ok {
        // 更新父任务的 Todo 列表
        parentTask := e.GetParentTask()
        parentTask.UpdateTodoItem(e.SubTaskID, "completed", 100)

        // 检查是否所有子任务都完成
        if parentTask.AllSubTasksCompleted() {
            Finish(parentTask.ID())
        }
    }
})
```

## 七、Todo 列表更新规则

### 7.1 触发更新的事件

| 事件 | 更新内容 |
|------|---------|
| `StartSubTask()` 调用 | 添加新 item，status=distributed |
| `Process(subID, x%)` | 更新对应 item 的 progress |
| `Finish(subID)` | 更新 status=completed, progress=100 |

### 7.2 自动完成判断

当父任务满足以下条件时，自动调用 `Finish(parentID)`：
- 所有子任务 status = "completed"
- 父任务本身状态 = running
