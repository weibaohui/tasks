# 文档编写规范

## 1. 文档类型

| 类型 | 说明 | 存放位置 |
|------|------|----------|
| 规范 (Standard) | 必须遵守的约束规则 | `standards/` |
| 指南 (Guide) | 操作参考文档 | `guides/` |

## 2. 文件命名

### 规范文件
```
序号-名称.md

示例：
01-document.md    # 文档编写规范
02-architecture.md # 技术架构规范
03-git-workflow.md # Git 工作流
```

### 指南文件
```
名称.md

示例：
debugging.md      # 问题排查指南
database.md       # 数据库指南
```

## 3. 编号规则

| 编号 | 类型 | 说明 |
|------|------|------|
| 01 | 基础 | 文档、架构基础 |
| 02 | 架构 | 技术架构 |
| 03 | 流程 | 开发流程 |
| 04 | 测试 | 测试规范 |
| 10+ | 专项 | 专项规范 |

## 4. 文档结构

### 4.1 规范文档结构

```markdown
# 标题

## 1. 原则/规则

## 2. 具体要求

## 3. 示例

## 4. 检查清单
```

### 4.2 指南文档结构

```markdown
# 标题

## 概述
什么是这个指南解决的问题

## 前置条件
开始之前需要什么

## 操作步骤
具体操作

## 常见问题
FAQ

## 参考链接
相关文档
```

## 5. 格式要求

### 5.1 标题
- 一级标题：`# 标题`
- 二级标题：`## 标题`
- 三级标题：`### 标题`

### 5.2 列表
- 无序列表：`- ` 或 `* `
- 有序列表：`1. ` `2. ` `3. `
- 任务列表：`- [ ] 待办` `- [x] 已完成`

### 5.3 代码
- 行内代码：`` `code` ``
- 代码块：``` ```lang ``` ... ``` ````

### 5.4 表格
```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

## 6. 编写原则

1. **简洁** - 避免冗余，一句话说清
2. **具体** - 提供具体示例，不是抽象描述
3. **可操作** - 给出明确步骤，不是模糊建议
4. **最新** - 保持文档与代码同步

## 7. 图片引用

```markdown
<!-- 相对路径引用 -->
![描述](./images/file.png)

<!-- 外部链接 -->
![描述](https://example.com/image.png)
```

## 8. 文档审核

- 规范文档变更需要 PR review
- 指南文档可以单独更新
- 重大变更需通知相关人员
