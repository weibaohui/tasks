# 心跳模板管理需求文档

## 背景

Pet Shop 项目的默认心跳包含 3 个任务（派发需求、处理 PR、提出优化点）。在将单一心跳拆分为多个独立心跳后，用户发现每次创建新心跳都需要手动复制粘贴 Prompt 内容，操作繁琐且容易出错。因此需要引入心跳模板管理功能，提升创建心跳的效率。

## 目标

1. 支持将当前心跳的 Prompt 内容和需求类型保存为可复用的模板。
2. 支持在创建/编辑心跳时，从已有模板中快速选择并自动填充表单。
3. 模板全局共享，不绑定具体项目。

## 功能需求

### FR-1 模板列表展示
- 系统提供全局心跳模板列表接口。
- 前端在心跳编辑弹窗中展示模板下拉选择框。

### FR-2 应用模板
- 用户选择某个模板后，系统自动将模板的 `md_content` 和 `requirement_type` 填充到当前心跳表单中。
- 填充后用户仍可手动修改。

### FR-3 保存为模板
- 用户在编辑心跳时，可将当前填写的内容保存为新模板。
- 保存时需要输入模板名称。
- 保存成功后，模板列表立即刷新，新模板在下拉框中可见。

### FR-4 删除模板
- 用户可以在模板管理页面或通过其他入口删除不再需要的模板（本迭代通过前端直接调用删除 API 支持）。

## 非功能需求

- 模板数据存储在 SQLite 中，表结构简洁。
- 模板不绑定项目 ID，所有项目共用。
- 后端编译通过，前端无 TypeScript 错误。
- 核心 domain/application 逻辑需有单元测试覆盖。
