# Agent 管理界面重构需求文档

## 背景

当前 Agent 管理界面将所有类型的 Agent 混在一个列表中展示，且左侧菜单平铺了 `Agents 管理`、`Skills 管理`、`MCP 管理` 三个一级菜单。随着 Agent 类型增加（个人助理、ClaudeCode、OpenCode、未来还会有 OpenClaw、Hermes 等），这种平铺结构会造成：
1. 菜单混乱，用户难以快速定位功能
2. Agent 列表混杂不同类型，信息密度和关注点不同
3. Skills、MCP 本质上是"个人助理"的专属配置资源，单独占一级菜单不合理

## 目标

1. 按 Agent 类型对列表进行分组展示
2. 将 Skills、MCP 从一级菜单移除，内嵌到个人助理的配置流程中
3. 新建 Agent 改为"先选类型，再填配置"的两步流程
4. 保持现有后端数据和 API 兼容，不改动领域模型

## 功能需求

### FR-1 左侧菜单重构
- `Skills 管理` 和 `MCP 管理` 从一级菜单移除
- `Agents 管理` 保留为一级菜单，名称可保持或简化为 `Agent 工坊`
- 原 `Skills`、`MCP` 管理功能需从个人助理编辑页内可直达

### FR-2 Agent 列表页类型分组
- Agent 列表顶部增加类型分段器（Segmented/Tabs）
- 支持的类型分组（初始）：`全部`、`个人助理`、`编程 Agent`
- 列表内容按所选分组过滤显示
- 每个分组支持独立的搜索/刷新操作

### FR-3 新建 Agent 两步流程
- 第一步：选择 Agent 类型（卡片式展示，带简要说明）
- 第二步：进入对应类型的配置表单
- 类型选择后不可在表单内随意切换（减少配置项混乱）

### FR-4 个人助理内嵌资源管理
- 在个人助理的编辑抽屉中，Skills 和 MCP 配置保持现有表单内容
- 增加快捷入口，可弹窗管理 Skills 列表和 MCP 绑定
- 或直接在"技能工具"标签页中提供"去管理 Skills"的链接，跳转至内嵌子页面

### FR-5 向后兼容
- 现有 Agent 数据（agent_type 字段）无需迁移
- 现有 API 端点保持不变
- 编辑现有 Agent 时直接进入对应类型的配置表单（无需重新选类型）

## 非功能需求

- 响应式：移动端类型分段器可横向滚动
- 性能：列表过滤在本地完成，不增加额外 API 请求
- 可扩展性：新增 Agent 类型时，仅需增加一个分组 Tab 和对应的类型卡片
