# 需求列表展示耗时 - 需求说明

## 背景

在“需求列表页面”中，当前仅展示了需求标题、类型、状态、Agent 状态、进度等信息，缺少从开始到结束的总耗时信息，导致排查执行效率与对比不同需求耗时不便。

## 目标

- 在需求列表（表格视图）中新增一列：展示需求“从开始到最后”的耗时。
- 耗时展示为紧凑格式，示例：`1h28m`、`3m`、`2d5h`。

## 口径与计算规则

- 优先使用需求维度字段：
  - 开始时间：`Requirement.started_at`
  - 结束时间：`Requirement.completed_at`
- 若需求维度字段缺失，回退到运行时字段：
  - 开始时间：`Requirement.agent_runtime.started_at`
  - 结束时间：`Requirement.agent_runtime.ended_at`
- 结束时间缺失时（需求正在运行中），以当前时间 `Date.now()` 作为“最后时间”。
- 开始时间缺失时，显示 `-`。
- 耗时为：`max(0, endedAt - startedAt)`，单位为毫秒。

## 展示规则

- 表格列名：`耗时`（或可扩展为 tooltip 说明“开始→结束”口径）。
- 输出格式：使用 `humanize-duration` 将毫秒格式化为紧凑形式（无空格、无逗号分隔）。
