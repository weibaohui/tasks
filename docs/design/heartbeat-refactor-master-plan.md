# 心跳系统重构总方案（前后端一体化）

## 1. 文档目标

本方案用于指导心跳相关能力的前后端重构，覆盖以下模块：

- 后端：心跳实例、心跳场景、心跳模板、调度触发、Webhook 事件触发
- 前端：项目内心跳管理、心跳场景管理、Webhook 绑定与事件日志展示
- 架构：统一触发链路、统一数据契约、统一可观测性

目标是将当前“可运行但分散”的实现，重构为“可维护、可追踪、可扩展”的自动化系统。

## 2. 重构原则

### 2.1 架构原则

- 单一职责：领域对象不承担传输层序列化约束，HTTP DTO 与 Domain 解耦
- 一致性优先：涉及批量替换（如场景应用）必须保障事务一致性
- 可观测优先：每次触发都要有可追踪的状态与结果
- 向后兼容：优先保留现有 API 路由，在服务内部平滑演进

### 2.2 前端原则

- 信息架构统一：将心跳实例、场景、Webhook 聚合到“自动化中心”
- 操作可预期：高风险操作必须先展示影响预览
- 状态可感知：统一加载态、成功态、失败态和空态
- 组件可复用：避免各页面独立造轮子

## 3. 当前核心问题清单

### 3.1 后端问题

- 场景接口直接绑定 Domain 结构，存在 JSON 字段契约不清风险
- 应用场景“先删后建”，缺少事务，失败时可能导致数据半完成
- 调度器运行期并发刷新调度条目，缺少并发保护
- 触发链路为“创建需求后派发”，失败恢复策略不足

### 3.2 前端问题

- 心跳相关入口分散，操作链路长，认知负担高
- 需求类型在不同页面硬编码不一致
- 场景应用缺少“变更预览”，风险不可见
- Webhook 事件与心跳执行结果没有统一视图

## 4. 总体目标状态

### 4.1 业务能力

- 支持统一入口管理自动化：实例、场景、事件绑定、执行历史
- 支持安全应用场景：具备影响评估、事务回滚、失败可重试
- 支持统一触发观测：定时/手动/Webhook 三类触发一致可见

### 4.2 工程能力

- 场景 API 使用明确 DTO（snake_case）+ 服务层映射
- 场景应用服务具备事务边界
- 前端拥有自动化中心页面骨架与统一状态组件基础

## 5. 分阶段实施计划

## 阶段 A：止血与契约统一（优先级 P0）

### A.1 后端任务

- 新增 HeartbeatScenario HTTP DTO（请求/响应）
- Handler 层完成 DTO -> Domain 映射与校验
- 场景应用逻辑改造为事务执行
- 事务失败时保证原子回滚，不产生半成功数据

### A.2 前端任务

- 创建“自动化中心”页面骨架（总览/实例/场景/Webhook 四分区）
- 菜单入口由“心跳场景”升级为“自动化中心”
- 保留原页面能力，通过新入口统一承载（过渡期）

### A.3 验收标准

- 自定义场景项字段保存与读取一致
- 场景应用失败后，心跳列表与绑定关系保持改造前状态
- 新入口可访问并可承载现有心跳操作

## 阶段 B：执行链路重构（优先级 P1）

### B.1 后端任务

- 引入 HeartbeatOrchestrator（统一触发入口）
- 调度触发、手动触发、Webhook 触发统一走 TriggerPipeline
- 触发执行结果结构化记录（成功/失败/错误原因）
- 调度器 entries 并发访问加锁

### B.2 前端任务

- 自动化中心展示“最近触发”和“最近执行结果”
- 增加执行详情抽屉（触发源、步骤、结果、错误）
- 统一失败提示与重试操作入口

### B.3 验收标准

- 三类触发链路行为一致
- 调度并发刷新无竞态告警（`go test -race`）
- 前端可查看每次执行结果

## 阶段 C：体验与可维护性升级（优先级 P2）

### C.1 后端任务

- 场景应用从“全量替换”演进为“差量更新”
- 补齐触发链路指标与日志字段
- 形成清晰的领域事件（可选）

### C.2 前端任务

- 心跳新建改为向导模式
- 场景应用前提供“影响预览”与二次确认
- 统一状态组件与空态组件
- 完成移动端基础适配

### C.3 验收标准

- 操作路径更短，用户可预测
- 页面内状态表达一致
- 移动端可正常完成核心流程

## 7. 风险与回滚策略

### 7.1 主要风险

- 场景事务改造涉及数据库操作路径变化
- 前端入口合并可能影响原有访问路径
- 执行链路统一可能引发历史逻辑兼容问题

### 7.2 降低风险措施

- 每阶段提交独立 PR，保证可回退
- 前端保留旧页面能力直到新页面稳定
- 后端在关键路径保留原日志，便于对比排查

## 8. 测试策略

### 8.1 后端

- 单元测试：DTO 映射、事务回滚、触发链路错误语义
- 集成测试：场景应用 -> 心跳列表 -> 调度刷新
- 并发测试：调度器刷新与触发并发执行

### 8.2 前端

- 关键交互测试：场景应用、心跳编辑、事件重触发
- 状态测试：加载、错误、空态展示
- 兼容测试：桌面端与移动端基础流程
