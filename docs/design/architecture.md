## 分层职责

### 1. Interface Layer (接口层)

**职责**：
- 接收外部请求
- 输入校验、DTO转换
- 调用应用服务
- 返回响应

**不包含**：
- 业务逻辑
- 事务控制

### 2. Application Layer (应用层)

**职责**：
- 编排领域对象完成用例
- 事务管理
- 发布领域事件
- 权限检查

**不包含**：
- 业务规则（如状态转换规则）
- 与具体存储技术相关的代码

### 3. Domain Layer (领域层)

**职责**：
- 核心业务逻辑
- 业务规则验证
- 领域事件定义
- 仓储接口定义

**特点**：
- 不依赖其他层
- 只依赖Go标准库
- 纯内存操作

### 4. Infrastructure Layer (基础设施层)

**职责**：
- 仓储实现
- 消息总线实现
- 外部服务调用
- 工具类实现

**依赖**：
- 实现领域层定义的接口

## 依赖关系

```
Interface Layer
       ↓
Application Layer
       ↓
Domain Layer (核心，不依赖其他层)
       ↑
Infrastructure Layer (依赖领域层接口)
```

**依赖倒置原则**：领域层定义接口，基础设施层实现接口。
